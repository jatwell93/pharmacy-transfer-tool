**Document structure — 5 numbered sections per D-14:**

**Section 1: Sell-Through Filter**
- Rule: destination store qualifies only if `destROU >= originSOH / SELL_THROUGH_LIMIT_MONTHS` where `SELL_THROUGH_LIMIT_MONTHS = 12`
- Worked example using SKU `PARA500`, originSOH = 240:
  - Store A: ROU = 25, minROU = 240/12 = 20, 25 >= 20 = INCLUDED
  - Store B: ROU = 15, 15 >= 20 = EXCLUDED
  - Store C: ROU = 20, 20 >= 20 = INCLUDED (boundary, inclusive `>=`)
- Django behavior: Same formula, hardcoded `sell_through_limit_months = 12`. Correct.
- TypeScript behavior: Same. No change. Keep as internal constant `SELL_THROUGH_LIMIT_MONTHS = 12`.

**Section 2: Months-Cover Cap Formula**
- Rule: `maxTransferQty = Math.max(0, monthsCoverTarget * destROU - destSOH)`. Actual transfer qty = `Math.min(originSOH, maxTransferQty)`. Exclude store if `maxTransferQty === 0`.
- Worked example (cap binds): monthsCoverTarget=3, destROU=20, destSOH=10: maxQty = max(0, 60-10) = 50, transfer = min(240, 50) = 50
- Worked example (dest over cap): monthsCoverTarget=3, destROU=20, destSOH=65: maxQty = max(0, 60-65) = 0, store EXCLUDED
- Worked example (no destSOH data): monthsCoverTarget=3, destROU=20, destSOH absent (defaults 0): maxQty = max(0, 60-0) = 60
- Django behavior: NO months-cover cap exists. All sell-through matches return full originSOH. BUG — receiving stores can become overstocked.
- TypeScript behavior: Cap enforced. monthsCoverTarget is REQUIRED parameter.

**Section 3: Ranged Sort Order**
- Rule: sort ranged-first (isRanged=true before false), then ROU descending within each group
- Worked example: Match A (isRanged=false, rou=30), Match B (isRanged=true, rou=10), Match C (isRanged=true, rou=25) => sorted: C, B, A
- Django behavior: `sort(key=lambda x: (-x['isRanged'], -x['rou']))`. Correct.
- TypeScript behavior: `a.isRanged !== b.isRanged ? (a.isRanged ? -1 : 1) : b.rou - a.rou`. Same logic.

**Section 4: is_ranged Parsing**
- Rule: truthy values (case-insensitive, after trim): `checked`, `yes`, `true`, `1`, `y`. All others = false.
- Table of examples: "Checked"->true, "YES"->true, "True"->true, "1"->true, "Y"->true, ""->false, "no"->false, "false"->false, "0"->false, undefined->false
- Django behavior: ONLY `"checked"` (case-insensitive) -> true. All others -> false. BUG — FRED exports may use "yes", "true", "1", or "y".
- TypeScript behavior: Set lookup against all 5 truthy variants. FIXED.
- Note: parsing happens at upload time (Phase 3), not at match time. `parseIsRanged` is a separate exported utility. `matchTransfers` receives `isRanged: boolean`.

**Section 5: NaN/Missing-Value Edge Cases**
- Rule: NaN/missing values are NOT silently defaulted to 0. Instead, DataQualityWarning emitted:
  - RouItem with NaN/missing rou: excluded from matching (rou <= 0 filter) + warning
  - DeadStockItem with NaN/missing soh: excluded from matching + warning
  - DeadStockItem with NaN/missing cost: INCLUDED in matching with cost=0 in result + warning (cost is display-only, not algorithmic)
- Worked example: rouItem { sku:'A', store:'B', rou:NaN, isRanged:false } => warning { sku:'A', field:'rou', reason:'rou is missing or non-numeric' }, item excluded
- Django behavior: `pd.to_numeric(...) or 0` silently converts NaN to 0. For ROU: accidentally correct exclusion (rou=0 filtered by rou>0). For cost: silently wrong (cost=0.0 with no warning). BUG.
- TypeScript behavior: Explicit Number.isNaN check. Warnings emitted. Use `Number.isNaN()` not `isNaN()`.

**Additional section — Algorithm Pipeline Overview:**
Before the 5 numbered sections, include a brief pipeline overview:
1. Filter deadStock to items with valid SKU and SOH > 0
2. Filter rouData to exclude originStore (case-insensitive) and items with rou <= 0
3. Index rouData by SKU for O(1) lookup
4. For each dead-stock item: find destination stores with matching SKU
5. Apply sell-through filter (section 1)
6. Apply months-cover cap (section 2), exclude stores with maxTransferQty = 0
7. Compute sellThrough = originSOH / destROU for each remaining match
8. Sort matches: ranged-first, then ROU-descending (section 3)
9. Return flat list of MatchResult objects + DataQualityWarning array
