## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|

**Semantic colors (informational — not accent):**

## Component Inventory

All components are new for Phase 3. No shadcn components. Build from Tailwind utilities matching existing codebase style.

### UploadPage

- Route: `/upload`
- Layout: `AppShell` wrapping a full-width `main` area with `p-8`
- Page title: `h1` "Upload Stores" — `text-2xl font-semibold text-[#0F172A] tracking-[-0.01em] mb-6` in Space Grotesk
- Header row: flex row with page title left, "Add Store" button right — `flex items-center justify-between mb-6`

### StoreCardGrid

- Layout: CSS grid `grid grid-cols-[repeat(auto-fill,minmax(280px,1fr))] gap-4`
- One `StoreCard` per store in org
- Empty state rendered when `stores.length === 0`

### StoreCard

- Container: `bg-white border border-[#E2E8F0] rounded-lg p-4 flex flex-col gap-3`
- Store name: `text-base font-semibold text-[#0F172A]` in Space Grotesk — display as "Balwyn (987)" if store number set, plain "Balwyn" if absent
- Incomplete indicator: amber dot `w-2 h-2 rounded-full bg-[#D97706]` + text "Missing files" in `text-[13px] text-[#D97706]` — visible only when one file is uploaded and the other is not
- ROU status row: `text-[13px]` label + status badge (see FileStatusBadge)
- Dead-stock status row: same pattern as ROU
- "Upload files" button: `mt-auto` — text button `text-[13px] text-[#0F766E] font-semibold min-h-[44px] px-3` with `hover:underline` — opens upload modal pre-filled with store data

### FileStatusBadge

Two states only — no intermediate states:

| State | Visual |
|-------|--------|
| Uploaded | Green dot `w-2 h-2 bg-[#10B981] rounded-full` inline + date string `text-[13px] text-[#475569]` e.g. "29 Mar 2026, 14:32" |
| Not uploaded | Em dash `–` + text "not uploaded" `text-[13px] text-[#94A3B8]` |

### AddStoreButton (primary CTA)

- Element: `<button>` — `bg-[#0F766E] text-white text-[13px] font-semibold rounded-md px-4 min-h-[44px] flex items-center gap-2 hover:bg-[#0D5D5A] transition-colors`
- Icon: `<Plus size={16} strokeWidth={1.5} aria-hidden="true" />` from lucide-react
- Label: "Add Store"
- Opens upload modal in new-store mode (store name/number fields are editable)

### UploadModal

- Overlay: `fixed inset-0 bg-[#0F172A]/60 flex items-center justify-center z-50` — clicking overlay dismisses modal
- Dialog: `bg-white rounded-xl shadow-xl w-full max-w-md mx-4 p-6 flex flex-col gap-6` — role="dialog", aria-modal="true", aria-labelledby pointing to modal title
- Focus trap: first focusable element on open is the Store Name field (new-store mode) or the ROU file input (existing-store mode)
- Title: `text-[20px] font-semibold text-[#0F172A] tracking-[-0.01em]` in Space Grotesk — "Add Store" (new) or "Upload Files — {Store Name}" (existing)
- Close button: `absolute top-4 right-4` — `<X size={16} />` from lucide-react, `min-h-[44px] min-w-[44px]` touch target, `aria-label="Close"`
- Store Name field: text input, required in new-store mode, read-only in existing-store mode — `text-base text-[#0F172A]`
- Store Number field: text input, optional — `text-base text-[#0F172A] placeholder-[#94A3B8]`
- ROU file picker: `<input type="file" accept=".csv,.xlsx">` with visible label "ROU Report" + help tooltip icon
- Dead-stock file picker: `<input type="file" accept=".csv,.xlsx">` with visible label "Dead-Stock Report" + help tooltip icon
- Help tooltip (ⓘ): `<Info size={14} />` from lucide-react, tooltip on hover/focus — see Copywriting Contract for tooltip content
- Inline error zone: below each file picker — `text-[13px] text-[#EF4444]` — hidden unless error present
- Replace-confirmation warning: shown when existing-store mode and at least one previously-uploaded file is being overwritten — amber/red banner above Upload button (see Copywriting Contract)
- Upload button: `bg-[#0F766E] text-white text-[13px] font-semibold rounded-md px-4 min-h-[44px] w-full` — disabled (with `opacity-40 cursor-not-allowed`) when no files are selected; enabled when at least one file is selected
- Cancel button: text button `text-[13px] text-[#475569] min-h-[44px] px-4 hover:text-[#0F172A]`
- Buttons: cancel left, upload right — `flex justify-end gap-3`

### UploadModal — Loading State

While upload is in progress:
- Upload button shows spinner icon `<Loader2 size={16} className="animate-spin" />` + "Uploading..." label — button is disabled
- File pickers and text fields are disabled (pointer-events-none + opacity-60)
- Modal is not closable (close button disabled, overlay click blocked)

## Accessibility Contract

- All interactive elements: `min-h-[44px]` touch target (pre-established in codebase)
- All icon-only buttons: `aria-label` attribute required (e.g. close button: `aria-label="Close"`)
- All decorative icons: `aria-hidden="true"` (pre-established in AppShell.tsx)
- Modal: `role="dialog"` + `aria-modal="true"` + `aria-labelledby="{modal-title-id}"`
- Focus trap inside open modal: Tab key cycles through focusable elements within modal only
- Escape key closes modal when no upload is in progress
- Status badges use text content (not color alone) to convey meaning — "not uploaded" text, not just a grey dot
- File inputs: associated `<label>` elements with `htmlFor`
- Error messages: `role="alert"` so screen readers announce inline errors immediately on appearance
- Skip link: already implemented in AppShell.tsx — do not remove
