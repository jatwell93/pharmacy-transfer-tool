# Phase 1 — UI Design Contract

> Visual and interaction contract for the Phase 1 Foundation scaffold.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope

Phase 1 delivers two screens only:

1. **Sign-in page** (`/sign-in`) — Clerk `<SignIn />` component styled with PharmIQ colours.
2. **App shell** (`/`) — Teal header, sidebar with disabled nav items, empty main content area with a placeholder message.

This contract does NOT cover file upload UI, match result tables, billing prompts, dark mode toggle, or PDF export — those are Phase 3, 4, 5, and 6 respectively.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap, badge padding, inline spacing between icon and label |
| sm | 8px | Nav item internal padding, compact element spacing |
| md | 16px | Default element padding, sidebar item padding, content gap |
| lg | 24px | Section padding, header height padding, card padding |
| xl | 32px | Layout column gap, sidebar width inner padding |
| 2xl | 48px | Major section breaks on the sign-in page |
| 3xl | 64px | Sign-in card max-width breathing room |

Exceptions:
- Sidebar width: 240px (fixed, not from scale — layout constraint)
- Header height: 56px (not from scale — standard app bar height)
- Touch targets: minimum 44px height on all interactive elements (nav items, buttons)

## Typography

Phase 1 uses 4 type roles. All data values (if any appear in future phases) must use `font-variant-numeric: tabular-nums`.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 16px | 400 | 1.6 | Inter | General text, placeholder copy, sidebar item labels |
| Label | 13px | 400 | 1.4 | Inter | Nav item text, sidebar footer links, form field labels |
| Heading | 24px | 600 | 1.3 | Space Grotesk | Sign-in page heading, dashboard section heading |
| Display | 32px | 600 | 1.2 | Space Grotesk | PharmIQ wordmark in header (if text-based fallback needed) |

Letter spacing:
- Heading and Display: `-0.01em`
- Body and Label: `0` (default)

Minimum body text: 16px enforced — avoids iOS auto-zoom on mobile inputs and meets brand-guidelines.md §3.3.

## Color

### Light mode (Phase 1 only — dark mode is Phase 6)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / `#F8FAFC` | Page background (`#FFFFFF`), sidebar background (`#F8FAFC`), card surfaces |
| Secondary (30%) | `#0F172A` (header) / `#F8FAFC` (sidebar) | Teal header uses navy base for sidebar, navy text on light surfaces |
| Accent (10%) | `#0F766E` (teal primary) | Header background, primary CTA button fill, active nav indicator, focus rings |
| Destructive | `#EF4444` | Sign out confirmation only (if destructive confirmation is needed); no other use in Phase 1 |

**Full token set (CSS custom properties to declare in `apps/web/src/index.css`):**

```css
:root {
  /* Brand */
  --color-teal:          #0F766E;
  --color-teal-light:    #14B8A6;
  --color-teal-dark:     #0D5D5A;
  --color-amber:         #D97706;
  --color-navy:          #0F172A;

  /* Surface */
  --color-surface:       #FFFFFF;
  --color-surface-gray:  #F8FAFC;
  --color-border-light:  #E2E8F0;
  --color-border-mid:    #CBD5E1;

  /* Text */
  --color-text-primary:  #0F172A;
  --color-text-secondary:#475569;
  --color-text-muted:    #94A3B8;

  /* Semantic */
  --color-success:       #10B981;
  --color-warning:       #D97706;
  --color-critical:      #EF4444;
  --color-info:          #0EA5E9;
}
```

**Amber is NOT used in Phase 1.** Amber is reserved for alerts, dead-stock warnings, and freemium prompts (Phases 3–5). Using amber in the Phase 1 scaffold would dilute its urgency signal before the relevant features exist.

Accent (`#0F766E`) is reserved for:
- Header/appbar background
- Primary CTA button fill (e.g. "Sign in" button on the Clerk sign-in page)
- Active/current state indicator on nav sidebar
- Focus ring on interactive elements

## Components

### Header

- Height: 56px
- Background: `#0F766E` (teal)
- Content: PharmIQ wordmark (white) on left, user avatar/name from Clerk `<UserButton />` on right
- Logo text: "PharmIQ" in Space Grotesk 600, white, with "IQ" optionally distinguished. Use text fallback if SVG logo is not available in Phase 1.
- No tagline in the header chrome (brand-guidelines.md §1.6: "In-app dashboard — omit in navigation chrome")

### Sidebar

- Width: 240px (fixed, left-anchored)
- Background: `#F8FAFC`
- Border right: `1px solid #E2E8F0`
- Nav items (Upload, Match, Billing): rendered as `<a>` tags with `aria-disabled="true"`, `pointer-events: none`, `opacity: 0.4`, `cursor: not-allowed`
- Active item indicator: 3px teal left border + teal text color on the active item (the `/` route has no active nav item in Phase 1 — all items are disabled)
- Nav item height: 44px minimum (touch target requirement)
- Nav item padding: 8px 16px (sm md)
- Nav item text: 13px Inter 400 (`--color-text-secondary` when disabled)
- Nav icons: Lucide React, 16px, stroke-width 1.5, same color as text

**Sidebar footer:**
- Separator: `1px solid #E2E8F0` above footer section
- Items: "Settings" (disabled, same disabled treatment as nav), "Sign out" (active, Clerk `<SignOutButton />`)
- "Sign out" is a destructive action: render in `--color-text-secondary`, separated spatially from navigation items by the separator above

### Main content area (Phase 1 placeholder)

- Padding: 32px (xl)
- Background: `#FFFFFF`
- Placeholder text: "Phase 1 scaffold — features coming soon" (see Copywriting Contract)
- Text style: Body 16px, `--color-text-muted`

### Sign-in page

- Layout: Centered card on `#F8FAFC` background
- Card max-width: 400px, padding: 32px, border-radius: 8px, border: `1px solid #E2E8F0`
- Clerk `<SignIn />` component with `appearance` prop applying PharmIQ teal as the primary colour
- Heading above card: "Sign in to PharmIQ" — Space Grotesk 600 24px `--color-text-primary`
- Sub-heading: "Dead-Stock Optimizer" — Inter 400 16px `--color-text-secondary`

**Clerk appearance configuration (minimal — full brand theming is Phase 6):**
```typescript
appearance={{
  variables: {
    colorPrimary: '#0F766E',
    colorText: '#0F172A',
    colorBackground: '#FFFFFF',
    fontFamily: 'Inter, system-ui, sans-serif',
  }
}}
```

### Organisation setup screen

After sign-in, if `auth.orgId` is absent, show a transitional screen before the app shell (per RESEARCH.md open question 1):

- Heading: "Set up your pharmacy group" — Space Grotesk 600 24px
- Body: "You need a pharmacy group to use Dead-Stock Optimizer." — Inter 400 16px
- CTA: Clerk `<CreateOrganization />` component
- Background: `#F8FAFC`, same centered card layout as sign-in

## Interaction States

| State | Treatment |
|-------|-----------|
| Teal button hover | `#0D5D5A` background (teal-dark), no layout shift |
| Teal button focus | 2px `#0F766E` outline, 2px offset (WCAG 2.1 AA visible focus) |
| Disabled nav items | `opacity: 0.4`, `cursor: not-allowed`, `pointer-events: none`, `aria-disabled="true"` |
| Sign out (active) | Normal text weight, `cursor: pointer`, no disabled treatment |
| Loading state | Inline spinner on the Clerk `<SignIn />` component (Clerk handles this natively) |

Transition duration: 150ms ease-out for hover color changes. No layout-shifting transitions.

## Accessibility Baseline

Minimum requirements for Phase 1:

- All interactive elements have visible focus rings (2px teal outline, 2px offset)
- All Lucide icons that are decorative have `aria-hidden="true"`
- All Lucide icons that convey meaning have an `aria-label` or adjacent visible label
- Disabled nav items use `aria-disabled="true"` (not `disabled` on `<a>` — use attribute)
- Sidebar `<nav>` has `aria-label="Main navigation"`
- Skip link: `<a href="#main-content" class="sr-only focus:not-sr-only">Skip to main content</a>` in `<header>`
- Colour contrast: `#0F172A` text on `#FFFFFF` background = 19.6:1 (AAA). `#FFFFFF` on `#0F766E` header = 4.6:1 (AA pass).
- Heading hierarchy: `<h1>` in sign-in card, `<h1>` in dashboard — one per page, no level skips
