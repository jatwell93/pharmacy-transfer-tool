## Spacing Scale

Declared values — multiples of 4 only (8-point scale, halved for tight UI):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gap inside badges, inline spacing between bullet metrics |
| sm | 8px | Spacing between pricing card header label and plan name, between limit rows |
| md | 16px | Internal card padding (horizontal), gap between usage stat items in the compact row |
| lg | 24px | Card padding (vertical), section gap between usage row and pricing card grid |
| xl | 32px | Gap between the pricing cards section and the "Manage subscription" link row |
| 2xl | 48px | Page top padding inside main content area |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: all interactive buttons must meet 44px minimum height (existing app convention — see AppShell SignOut button min-h-[44px]).
- Pricing card width: cards use equal-width flex children inside a `flex gap-6` row — no fixed pixel width; each card stretches to 1/3 of the available content area.
- "Current plan" badge: 4px vertical padding, 8px horizontal padding — compact inline badge.

## Color

All values drawn from `apps/web/src/index.css` CSS custom properties. Dark mode handled automatically via `:root.dark` token flips.

| Role | Light value | Dark value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#FFFFFF` (`--color-surface`) | `#0F172A` (`--color-surface`) | Page background, modal background |
| Secondary (30%) | `#F8FAFC` (`--color-surface-gray`) | `#1E293B` (`--color-surface-gray`) | Pricing card backgrounds, usage row strip, compact metric bar |
| Accent (10%) | `#0F766E` (`--color-teal`) | `#0F766E` (unchanged) | See accent reserved list below |
| Destructive | `#EF4444` (`--color-critical`) | `#EF4444` (unchanged) | Session fetch failure error state only |

Accent (`#0F766E`) reserved for:
1. Current-plan card border — 2px solid `#0F766E` (D-03)
2. "Current plan" badge background inside the highlighted card (D-03)
3. Checkout confirming spinner color while loading (D-10)
4. "Upgrade to Pro" / "Upgrade to Enterprise" CTA buttons on non-current cards — teal fill, white label
5. Focus ring on all interactive elements (`focus-visible:outline-[var(--color-teal)]`)

Amber (`#D97706` / `--color-amber`) usage — NOT accent, but existing semantic warning color:
- Upgrade CTA on the free card ONLY IF the user is already on free tier and no teal CTA is visible on the same card (maintain existing MatchPage upgrade modal amber convention)
- Toast notification background for upgrade confirmation: amber is NOT used here — toast uses teal success pattern

### Pricing Card (per card)

- Container: `rounded-xl border-2 bg-[var(--color-surface-gray)] p-6 flex flex-col gap-4 flex-1`
- Current plan card: `border-[#0F766E]`; other cards: `border-[var(--color-border-light)]`
- "Current plan" badge (current card only): `inline-flex items-center rounded-full bg-[#0F766E] text-white text-[12px] font-semibold px-3 py-1 mb-2`
- Plan name: 20px Space Grotesk semibold, `--color-text-primary`
- Price: `text-[20px] font-semibold text-[var(--color-text-primary)]` + `text-[13px] font-normal text-[var(--color-text-secondary)]` for "/mo AUD" suffix
- Limits block: two rows, 13px Inter regular, `--color-text-secondary`, `gap-1`
- CTA button: `w-full min-h-[44px] rounded-md text-[13px] font-semibold` — teal fill for upgrade CTAs, disabled/label-only for current tier card

### Checkout Success State (D-10, D-11, D-12)

When `?checkout=success` query param is detected on mount:

1. **Loading state:** Replace pricing cards with a centred spinner (Loader2 from lucide-react, size 24, `--color-teal` color) + "Confirming your upgrade..." at 13px Inter muted.
2. **Success state:** Dismiss spinner, render normal BillingPage with new plan highlighted, AND show UpgradeToast above the pricing cards for 3 seconds.
3. **Failure state:** Replace spinner with SessionFetchError block — red-border card with message and Retry button.

### Toast (UpgradeToast)

- Fixed position: `fixed top-4 right-4 z-50`
- Container: `rounded-lg bg-[#0F766E] text-white px-5 py-4 flex items-center gap-3 shadow-lg max-w-sm`
- Text: `text-[13px] font-medium Inter`
- Dismiss button: `text-white/70 hover:text-white` — X icon (lucide-react X, size 16)
- Auto-dismiss: setTimeout 3000ms; clears on unmount

### Upgrade Modal (modified — MatchPage)

Existing modal structure preserved. Copy changes per `upgrade_to` value:

| `upgrade_to` | Title | Body | CTA label |
|---|---|---|---|
| `'pro'` | "Upgrade to Pro" | "Get 10 match runs/month and up to 10 stores for $10/mo AUD." | "Upgrade to Pro" |
| `'enterprise'` | "Upgrade to Enterprise" | "Unlimited match runs and stores for $100/mo AUD." | "Upgrade to Enterprise" |
| `undefined` (fallback) | "You've used your free run for this month" | "Upgrade to PharmIQ Pro for unlimited match runs." | "Upgrade to Pro" |

Modal overlay: `rgba(15, 23, 42, 0.5)` — existing convention from MatchPage.
Modal container: `rounded-xl p-8 max-w-sm` — existing convention.

## Accessibility Contract

All elements follow the existing app accessibility conventions established in AppShell and MatchPage:

- Touch targets: min-height 44px on all interactive elements (button, link)
- Focus rings: `focus-visible:outline-2 focus-visible:outline-offset-2` on all interactive elements
- Toast: `role="status" aria-live="polite"` — screen readers announce upgrade confirmation
- Toast dismiss button: `aria-label="Dismiss notification"` on the X button
- Upgrade modal: existing `role="dialog" aria-modal="true" aria-labelledby` pattern preserved
- Pricing cards: plan name `<h2>` inside each card; card container is not a button — only the CTA button inside is interactive
- "Current plan" badge: visually distinguished AND uses screen-reader-visible text "Current plan" (not just a border color change)
- Spinner: `aria-label="Confirming upgrade"` on Loader2 in checkout confirming state

---

## Dark Mode

All color values use CSS custom properties from `--color-*` tokens. Dark mode is implemented via `.dark` class on `document.documentElement` (see `index.css`). All surface, border, and text tokens flip automatically.

Exception: teal `#0F766E` and amber `#D97706` are brand constants — they do not change between light and dark mode. Card border accent color remains `#0F766E` in both modes. This matches the existing convention in MatchPage and AppShell.
