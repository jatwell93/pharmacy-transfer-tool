## Component Contract: Table Grid Update

This phase's entire UI surface is the existing match results table in `MatchPage.tsx`. The contract below defines the exact diff to the table structure.

### Grid Template (D-10)

**Before (current):**
```
grid-cols-[36px_1fr_1.2fr_1fr_1fr_100px_80px_100px_100px]
```
9 columns: expand | SKU | Description | Source Store | Destination Store | Qty | Dest ROU | Months Cover | Sell-Through

**After (Phase 16):**
```
grid-cols-[36px_1fr_1.2fr_1fr_60px_1fr_1fr_100px_80px_100px_100px]
```
11 columns: expand | SKU | Description | Department | Ranged | Source Store | Destination Store | Qty | Dest ROU | Months Cover | Sell-Through

This grid string must be applied in exactly 3 places in MatchPage.tsx (lines ~479, ~513, ~552):
1. Sticky header row div
2. Main result row div (type: 'result')
3. Sub-match row div (type: 'subMatch')

### Column Widths (D-09)

| Column | Width | Behaviour |
|--------|-------|-----------|
| Department | `1fr` | Flex. Truncates with `truncate` (overflow-hidden + text-ellipsis). No tooltip. |
| Ranged | `60px` | Fixed. Holds a single character (`✓` or `—`). |

### Column Header Text (D-08)

Exact header labels in `columnHeaders` array (line ~240):

```ts
const columnHeaders = [
  'SKU',
  'Description',
  'Department',   // NEW — full word, not "Dept"
  'Ranged',       // NEW
  'Source Store',
  'Destination Store',
  'Qty to Transfer',
  'Dest ROU',
  'Months Cover',
  'Sell-Through Time',
];
```

No tooltip on either new header. The `✓`/`—` convention is self-explanatory; the Ranged header label makes it clear.

### Ranged Cell Rendering (D-06, D-07)

**Main result rows only:**

| `isRanged` value | Rendered text |
|-----------------|---------------|
| `true` | `✓` |
| `false` | `—` |

Rendered as plain text inside the cell div. No icon component. No colored background. No pill.

```tsx
{/* Ranged */}
<div className="px-3 text-[13px] text-[var(--color-text-secondary)]">
  {result.isRanged ? '✓' : '—'}
</div>
```

**Sub-match rows:** Ranged cell renders empty (`null` or empty string `''`). The Ranged value belongs to the dead stock item (source), not the destination. It is not repeated on sub-rows.

### Department Cell Rendering (D-02, D-03)

**Main result rows:**

```tsx
{/* Department */}
<div className="px-3 text-[13px] text-[var(--color-text-secondary)] truncate">
  {result.department}
</div>
```

- When `department` is `""` (empty string — column absent from upload), the cell is blank. No dash, no "N/A", no placeholder text.
- When `department` has a value, it truncates with ellipsis if overflow. No tooltip. Matches Description column behaviour exactly.

**Sub-match rows:** Department cell renders empty (`''`). No value, no placeholder. Uses muted text style to stay consistent with other sub-row cells.

```tsx
{/* Department — empty on sub-rows */}
<div className="px-3 text-[13px] text-[var(--color-text-muted)] truncate" />
```

## Interaction Contract

### Existing interactions — unchanged

All existing table interactions (row expand/collapse, store toggle, preset buttons, Run Match, Export PDF, upgrade modal) are unchanged. Phase 16 touches only the column structure of the table.

### New interaction surface

**Department column:** No interaction. Read-only text. Truncates silently.

**Ranged column:** No interaction. Read-only text (`✓` or `—`).

**Row expansion (hasMultiple rows):** The expand/collapse behaviour is unchanged. When a row is expanded, sub-match rows appear below. Sub-match rows show blank Department and Ranged cells — this is intentional (D-07) and requires no user explanation.

### Focus / Accessibility

- New cells are `div` elements (not interactive). No `tabIndex`, no `role` attribute needed.
- Column headers are already non-interactive divs. The two new headers follow the same pattern.
- `truncate` class on Department cells means screen readers will read the full text value before truncation (the DOM contains the full string; CSS truncates visually only). No additional aria-label needed.
- The `✓` character (U+2713 CHECK MARK) and `—` character (U+2014 EM DASH) are both readable by screen readers. No `aria-label` override needed.
