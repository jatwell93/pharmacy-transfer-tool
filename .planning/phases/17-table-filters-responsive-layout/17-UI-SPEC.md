## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Phase 17 specific — filter strip padding:** `p-3` (12px) to match the store selector row (`p-3 bg-[var(--color-surface-gray)] rounded-lg border border-[var(--color-border-light)]`).

**Phase 17 specific — filter control gaps:** `gap-3` between label and control; `gap-4` between individual filter controls in the strip.

**Phase 17 specific — table cell padding:** `px-3` (12px horizontal) on all table cells — unchanged from Phase 16.

**Touch targets:** `min-h-[44px]` on the Run Match button (unchanged). Filter controls (native `<select>`, number input) use `min-h-[36px]` — consistent with preset buttons in control bar. The department dropdown trigger button uses `min-h-[36px]`.

**Responsive scroll:** `overflow-x: auto` on the outermost table container (`div.border.border-[var(--color-border-light)].rounded-lg`). Minimum column widths are enforced via the existing fixed/fr grid template; no additional min-width CSS needed beyond what the grid provides.

Exceptions: none beyond Phase 16 establishments.

## Component Contract: Filter Strip

### Placement (D-05, D-06)

The filter strip is inserted between the store selector row and the table/empty-state area. It uses the same container pattern as the store selector:

```tsx
{/* Filter strip — only rendered when hasRun && results.length > 0 */}
{hasRun && results.length > 0 && (
  <div
    className="flex flex-wrap items-center gap-4 mb-4 p-3 bg-[var(--color-surface-gray)] rounded-lg border border-[var(--color-border-light)]"
    style={{ fontFamily: "'Inter', system-ui, sans-serif" }}
  >
    {/* Filter controls — see each section below */}
    {/* Spacer */}
    <div className="ml-auto flex items-center gap-3">
      {/* Showing X of Y — only when filtered */}
      {/* Clear all — only when any filter non-default */}
    </div>
  </div>
)}
```

### Ranged Filter (D-10, TABLE-04)

Control type: native `<select>`.

```tsx
<div className="flex items-center gap-2">
  <label
    htmlFor="filter-ranged"
    className="text-[13px] font-medium text-[var(--color-text-secondary)]"
  >
    Ranged
  </label>
  <select
    id="filter-ranged"
    value={rangedFilter}
    onChange={e => setRangedFilter(e.target.value as 'all' | 'ranged' | 'non-ranged')}
    className="rounded-md border border-[var(--color-border-light)] px-2 py-1 text-[13px] text-[var(--color-text-primary)] bg-[var(--color-surface)] focus:outline-none focus:ring-1 focus:ring-[var(--color-teal)]"
  >
    <option value="all">All</option>
    <option value="ranged">Ranged only</option>
    <option value="non-ranged">Non-ranged only</option>
  </select>
</div>
```

State variable: `rangedFilter: 'all' | 'ranged' | 'non-ranged'`, default `'all'`.

Non-default condition for "Clear all" trigger: `rangedFilter !== 'all'`.

### Department Filter (D-11, TABLE-05)

Control type: custom button + positioned `<div>` dropdown with checkboxes. Not a native `<select multiple>` (per D-11).

**Trigger button:**

```tsx
<div className="relative">
  <button
    type="button"
    onClick={() => setDeptDropdownOpen(prev => !prev)}
    className="flex items-center gap-1 rounded-md border border-[var(--color-border-light)] px-2 py-1 text-[13px] text-[var(--color-text-secondary)] bg-[var(--color-surface)] hover:bg-[var(--color-surface-gray)] focus:outline-none focus:ring-1 focus:ring-[var(--color-teal)]"
  >
    {selectedDepts.size === 0 ? 'Department' : `Dept (${selectedDepts.size})`}
    <ChevronDown size={12} aria-hidden="true" />
  </button>
  {/* Dropdown panel — see below */}
</div>
```

**Button label rules:**

| `selectedDepts.size` | Button label |
|---------------------|--------------|
| 0 | `Department` |
| N (any N > 0) | `Dept (N)` |

**Dropdown panel:**

```tsx
{deptDropdownOpen && (
  <div
    className="absolute top-full left-0 mt-1 z-10 bg-[var(--color-surface)] border border-[var(--color-border-light)] rounded-md shadow-sm min-w-[160px] max-h-[200px] overflow-y-auto py-1"
  >
    {uniqueDepartments.map(dept => (
      <label
        key={dept}
        className="flex items-center gap-2 px-3 py-1.5 text-[13px] text-[var(--color-text-primary)] hover:bg-[var(--color-surface-gray)] cursor-pointer"
      >
        <input
          type="checkbox"
          checked={selectedDepts.has(dept)}
          onChange={() => {
            setSelectedDepts(prev => {
              const next = new Set(prev);
              next.has(dept) ? next.delete(dept) : next.add(dept);
              return next;
            });
          }}
          className="accent-[var(--color-teal)]"
        />
        {dept}
      </label>
    ))}
  </div>
)}
```

**Close-on-outside-click:** `useEffect` with `document.addEventListener('mousedown', handler)` — checks if click target is outside the dropdown `ref`. Cleanup on unmount.

State variables:
- `deptDropdownOpen: boolean`, default `false`
- `selectedDepts: Set<string>`, default `new Set()`

`uniqueDepartments`: derived via `useMemo` from `results.map(r => r.department).filter(Boolean)` deduplicated with `Array.from(new Set(...)).sort()`.

Non-default condition for "Clear all" trigger: `selectedDepts.size > 0`.

### Store Filter (D-12, TABLE-06)

Control type: native `<select>` with involves-store logic.

```tsx
<div className="flex items-center gap-2">
  <label
    htmlFor="filter-store"
    className="text-[13px] font-medium text-[var(--color-text-secondary)]"
  >
    Store
  </label>
  <select
    id="filter-store"
    value={storeFilter}
    onChange={e => setStoreFilter(e.target.value)}
    className="rounded-md border border-[var(--color-border-light)] px-2 py-1 text-[13px] text-[var(--color-text-primary)] bg-[var(--color-surface)] focus:outline-none focus:ring-1 focus:ring-[var(--color-teal)]"
  >
    <option value="">All stores</option>
    {uniqueStores.map(store => (
      <option key={store} value={store}>{store}</option>
    ))}
  </select>
</div>
```

`uniqueStores`: derived via `useMemo` from the union of `results.map(r => r.sourceStore)` and `results.map(r => r.bestMatch.store)`, deduplicated and sorted alphabetically.

Filter predicate (involves-store): `result.sourceStore === storeFilter || result.bestMatch.store === storeFilter`.

State variable: `storeFilter: string`, default `''` (empty string = All stores).

Non-default condition for "Clear all" trigger: `storeFilter !== ''`.

### Min Units Filter (D-14, D-15, D-16, TABLE-07)

Control type: number input. Label is "Min units" — no dollar sign anywhere (D-16 deviation from TABLE-07).

```tsx
<div className="flex items-center gap-2">
  <label
    htmlFor="filter-min-units"
    className="text-[13px] font-medium text-[var(--color-text-secondary)]"
  >
    Min units
  </label>
  <input
    id="filter-min-units"
    type="number"
    min={0}
    step={1}
    value={minUnits}
    onChange={e => {
      const v = parseInt(e.target.value, 10);
      if (!isNaN(v) && v >= 0) setMinUnits(v);
      else if (e.target.value === '') setMinUnits(0);
    }}
    className="w-16 rounded-md border border-[var(--color-border-light)] px-2 py-1 text-[13px] text-center text-[var(--color-text-primary)] focus:outline-none focus:ring-1 focus:ring-[var(--color-teal)]"
    aria-label="Minimum units to transfer"
  />
</div>
```

Same class pattern as `monthsCoverTarget` input. Filter predicate: `result.bestMatch.qtyToTransfer >= minUnits`.

State variable: `minUnits: number`, default `0`.

Non-default condition for "Clear all" trigger: `minUnits > 0`.

### Result Counter and Clear All (D-07, D-08)

Both live in the right side of the filter strip (`ml-auto` container):

```tsx
<div className="ml-auto flex items-center gap-3">
  {/* Showing X of Y — only when any filter is active */}
  {anyFilterActive && filteredResults.length !== results.length && (
    <span className="text-[13px] text-[var(--color-text-muted)]">
      Showing {filteredResults.length} of {results.length} results
    </span>
  )}
  {/* Clear all — only when any filter is non-default */}
  {anyFilterActive && (
    <button
      type="button"
      onClick={handleClearFilters}
      className="text-[13px] text-[var(--color-teal)] hover:underline"
    >
      Clear all
    </button>
  )}
</div>
```

`anyFilterActive`: `rangedFilter !== 'all' || selectedDepts.size > 0 || storeFilter !== '' || minUnits > 0`.

`handleClearFilters`: resets all four filter state variables to their defaults simultaneously.

## Component Contract: Responsive Table (D-01 to D-04)

### Horizontal Scroll Wrapper

The outer table container must gain `overflow-x: auto` while retaining the existing `border` and `rounded-lg` classes:

```tsx
<div
  className="border border-[var(--color-border-light)] rounded-lg overflow-hidden"
  style={{ overflowX: 'auto' }}
>
  {/* Sticky header + virtualized body — see below */}
</div>
```

### Sticky Header

The sticky header `div` must be wrapped in a container that scrolls horizontally in sync with the body. The header row itself uses `position: sticky; top: 0; z-index: 10` to stay pinned vertically. Horizontal sync is automatic when both live inside the same `overflow-x: auto` scroll container.

```tsx
<div
  className="bg-[var(--color-surface-gray)] border-b border-[var(--color-border-light)]"
  style={{ position: 'sticky', top: 0, zIndex: 10 }}
>
  <div className="grid grid-cols-[36px_1fr_1.2fr_1fr_60px_1fr_1fr_100px_80px_100px_100px] items-center min-w-[900px]">
    {/* ...column headers */}
  </div>
</div>
```

`min-w-[900px]` on the grid div ensures columns do not collapse below their readable minimum at narrow viewports. The outer `overflow-x: auto` wrapper absorbs the overflow and shows a scrollbar.

### Sticky SKU and Description Columns (D-02)

SKU (column 2) and Description (column 3) are sticky-left so they remain visible while the user scrolls right.

**Result row — SKU cell:**
```tsx
<div
  className="px-3 text-[13px] font-medium text-[var(--color-text-primary)] truncate"
  style={{ position: 'sticky', left: 36, zIndex: 1, background: 'var(--color-surface)' }}
>
  {result.sku}
</div>
```

**Result row — Description cell:**
```tsx
<div
  className="px-3 text-[13px] text-[var(--color-text-secondary)] truncate"
  style={{
    position: 'sticky',
    left: 36 + /* SKU column computed width */ 0,  /* executor resolves via ref or fixed calc */
    zIndex: 1,
    background: 'var(--color-surface)'
  }}
>
  {result.description}
</div>
```

**Executor note:** The sticky `left` offset for Description must account for the SKU column width. Since SKU is `1fr`, use a `ref`/`ResizeObserver` to measure at runtime, or set a fixed pixel minimum (e.g. `left: calc(36px + 120px)`) if a fixed min-width is set on the SKU column. The contract requires the executor to verify that Description does not overlap SKU at 768px — whichever technique achieves this is acceptable.

**Header row — SKU and Description headers:** apply the same `position: sticky`, `left` offsets, and `background: var(--color-surface-gray)` so the sticky header cells also show the correct background when scrolled.

**Sub-match rows:** same sticky treatment on SKU and Description cells (which show parent values), with `background: var(--color-surface)` (sub-rows use surface, not surface-gray).

### Virtualization Continuity (from CONTEXT.md)

Phase 17 inserts a filtering layer between `results` and `flatItems`. The new data flow is:

1. `filteredResults = useMemo(...)` — derived from `results` + all four filter state variables
2. `flatItems = useMemo(...)` — derived from `filteredResults` + `expandedRows` (unchanged logic, new input)

The scroll container retains `overflowY: 'auto'` alongside the new `overflowX: 'auto'`. Both overflow axes live on the same wrapper element.

`expandedRows` state is NOT reset when filters change — rows that are expanded remain expanded if they pass the filter; they are simply absent from `visibleItems` if they are filtered out.

## Interaction Contract

### Filter Interactions

| Control | Trigger | Effect |
|---------|---------|--------|
| Ranged `<select>` | `onChange` | Live filter — `filteredResults` recomputes immediately |
| Department trigger button | `onClick` | Toggles `deptDropdownOpen` |
| Department checkbox | `onChange` | Adds/removes dept from `selectedDepts`; live filter |
| Department dropdown | click outside | Closes dropdown (`mousedown` document listener) |
| Store `<select>` | `onChange` | Live filter — `filteredResults` recomputes immediately |
| Min units `<input>` | `onChange` | Live filter — integer ≥ 0 only; empty resets to 0 |
| Clear all | `onClick` | Resets all four filters to defaults simultaneously |

**No "Apply" button.** All filtering is live (D-09).

**AND logic:** All active filters are applied simultaneously. A row must satisfy every active filter to appear in `filteredResults` (D-09).

### Responsive Scroll

| Viewport | Behaviour |
|----------|-----------|
| Desktop (> 900px) | No horizontal scroll; table fills container normally |
| Tablet (768px) | Horizontal scroll via `overflow-x: auto`; SKU + Description sticky-left; no column hiding |
| Mobile (< 640px) | Same as tablet — one pattern for all sub-desktop (D-03) |

### Focus and Accessibility

- All native `<select>` and `<input>` elements: `id` + `<label htmlFor>` pairings. No `aria-label` override needed when a visible label exists.
- Department trigger button: `type="button"` to prevent form submission. No `aria-haspopup` or `aria-expanded` required for MVP but acceptable if added.
- Department dropdown items: `<label>` + `<input type="checkbox">` — natively keyboard-navigable and screen-reader-friendly.
- "Clear all" button: `type="button"`. No confirmation needed — filters reset immediately; the action is fully reversible.
- Sticky columns: no additional ARIA. The `position: sticky` is a visual affordance only; screen readers read the full table left-to-right.
- Result counter span: no `role="status"` required for MVP; the count updates live but is not critical for screen reader announcements in this use case.
- `min-h-[36px]` on all filter controls provides adequate touch target for tablet use (D-03).
