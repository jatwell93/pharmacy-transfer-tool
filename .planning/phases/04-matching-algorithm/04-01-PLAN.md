CRITICAL SCHEMA NOTE — `rou_data` table does NOT have an `is_ranged` column:
```sql
-- From apps/worker/src/db/schema.sql
CREATE TABLE IF NOT EXISTS rou_data (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      TEXT NOT NULL,
  store_id    UUID NOT NULL REFERENCES stores(id) ON DELETE CASCADE,
  sku         TEXT NOT NULL,
  description TEXT,
  rou         DOUBLE PRECISION,
  soh         DOUBLE PRECISION,
  uploaded_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE IF NOT EXISTS dead_stock (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id      TEXT NOT NULL,
  store_id    UUID NOT NULL REFERENCES stores(id) ON DELETE CASCADE,
  sku         TEXT NOT NULL,
  description TEXT,
  soh         DOUBLE PRECISION,
  is_ranged   BOOLEAN NOT NULL DEFAULT false,
  uploaded_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

The `isRanged` field on `RouItem` MUST default to `false` when constructing from rou_data rows because the rou_data table has no is_ranged column. The dead_stock table has is_ranged but that applies to the origin store's knowledge, not the destination. This means sort ordering by ranged status uses `false` for all destination matches from ROU data alone.
