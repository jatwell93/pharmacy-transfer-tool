# Milestone v1.1: Reporting & Tiered Billing

**Status:** ✅ SHIPPED 2026-05-13
**Phases:** 11–15
**Total Plans:** 8

## Overview

v1.1 delivered visual reporting and a 3-tier billing model for the PharmIQ Stock Transfer platform. Pharmacy managers now see dead stock unit breakdowns as charts before and after match runs, can calculate their inventory value exposure via the Cost Report panel, and the platform supports Free, Pro, and Enterprise pricing tiers enforced server-side via atomic Postgres counters.

## Milestone Summary

**Key Decisions:**

- `cost_ex` typed as DOUBLE PRECISION to match existing soh column type — enables aggregation without type coercion
- `plan_tier NOT NULL DEFAULT 'free'` — existing rows receive free tier automatically without backfill
- `hasCostData` as explicit signal (not `totalValue === 0`) — unambiguous COST-04 flag; store can have `totalValue === 0` with legitimate zero-cost items
- Synchronous checkout confirmation endpoint (`GET /billing/checkout-session/:sessionId`) eliminates webhook timing race on upgrade redirect (BILLING-09)
- In-place subscription update for Pro→Enterprise avoids duplicate subscription item billing (BILLING-08)
- Webhook idempotency via `stripe_event_id` deduplication table — `INSERT ON CONFLICT DO NOTHING RETURNING id` pattern
- Atomic match counter pattern reused from v1.0 (same `UPDATE ... WHERE count < limit RETURNING count`) — no regression

**Issues Resolved:**

- BILLING-08: Pro→Enterprise upgrade created duplicate subscription items — resolved by detecting existing subscription and passing subscription item ID to update call
- BILLING-09: Upgrade confirmation had webhook timing race (async webhook ~1–5s after redirect) — resolved by synchronous session fetch before rendering
- Phase 12: NEON HTTP driver serializes PG boolean as string — resolved with `Boolean()` defensive coercion in summary route

**Issues Deferred:**

- Phase 11 VERIFICATION.md missing — schema-only phase, SUMMARY confirms live NEON migration; accepted as documentation tech debt
- Phase 15 VERIFICATION.md missing — UAT.md 9/9 provides sufficient evidence; accepted as documentation tech debt
- Nyquist VALIDATION.md files for all 5 phases remain in `status: draft` — no functional impact; accepted as tech debt for v2 workflow improvement

**Technical Debt Incurred:**

- `Boolean()` defensive coercion in dead-stock-summary.ts — covers `"true"` case; `Boolean("false") === true` is a known limitation; acceptable given PG driver behavior returns actual boolean in practice
- PostMatchChart Before/After bars show "Units Out/In" framing (not literal before/after SOH) — conscious design decision; before/after SOH framing impractical for destination stores with 0 dead stock
