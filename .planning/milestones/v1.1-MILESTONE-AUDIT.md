### Requirements — Integration Wiring

### Integration Warnings (Non-Blocking)

1. **VIZ-02 framing:** `PostMatchChart` renders "Units Out/In" (not literal before/after SOH). Conscious design decision documented in code — before/after framing was impractical for destination stores with 0 dead stock. Functionally correct.

2. **BILLING-07 grace period is implicit:** Store gate fires at match time. Existing Free orgs with >3 stores are not blocked from prior uploads, but their next match run will hit the store gate and show an upgrade prompt. Upload history is preserved (not deleted). This matches the spec intent ("applies to new match runs only, not to historical upload data").

3. **MatchResult.cost not in results table:** `cost` field consumed by `CostReport` only, not rendered as a column in the match results table. Design choice — cost is a summary metric.
